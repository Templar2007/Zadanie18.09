Zadanie 9: Opis stylu zewnętrznego

  1. Składnia teoretyczna

    <link rel="stylesheet" type="text/css" href="nazwa_pliku.css">

  2. Gdzie wstawiany jest styl zewnętrzny, jaki znacznik jest za to odpowiedzialny oraz ile razy może on występować w dokumencie, jako podłączenie stylu CSS?

    Styl zewnętrzny jest podłączany w sekcji nagłówka dokumentu HTML za pomocą znacznika ``. Może on występować wielokrotnie w dokumencie, łącząc go z różnymi arkuszami styli.

  3. Jaka jest najprostsza forma wstawiania stylu zewnętrznego?

    <link rel="stylesheet" href="nazwa_pliku.css">

  4. Ile arkuszy styli może być dołączanych do jednego dokumentu HTML oraz jaka zasad musi być zachowana przy większej ilości dołączanych styli?

    Do jednego dokumentu HTML może być dołączonych wiele arkuszy styli. Musi być zachowana kolejność dołączania arkuszy styli - style z późniejszych arkuszy mogą nadpisywać style z wcześniejszych arkuszy.

  5. Co się dzieje w przypadku konfliktów?

    W przypadku konfliktów, gdy te same style są definiowane w różnych arkuszach styli podłączonych do tego samego dokumentu, decyduje zasada kaskadowości CSS. Style z późniejszych arkuszy mogą nadpisywać style z wcześniejszych arkuszy.

  6. Jak są tworzone i edytowane pliki arkuszy z użyciem edytora tekstowego?

    Pliki arkuszy styli mogą być tworzone i edytowane przy użyciu dowolnego edytora tekstowego, takiego jak Notepad++, Sublime Text, Visual Studio Code, itp.

  7. Jak są tworzone i edytowane pliki arkuszy z użyciem specjalnego edytora CSS?

    Pliki arkuszy styli mogą być tworzone i edytowane także za pomocą specjalistycznych edytorów CSS, takich jak Adobe Dreamweaver, Atom, czy Brackets.

  8. Jakie rozszerzenie posiadają pliki będące zewnętrznym arkuszem styli?

    Pliki będące zewnętrznymi arkuszami styli mają zazwyczaj rozszerzenie `.css`.

  9. Jakie są zalety zewnętrznego arkusza styli?

    1. Możliwość ponownego wykorzystania styli na wielu stronach internetowych.
    2. Poprawa czytelności kodu HTML i CSS poprzez separację struktury HTML od prezentacji.

  10. Uwagi na temat pisania w CSS

    Pisząc w CSS należy dbać o czytelność i zgodność z zasadami kaskadowości oraz selektorów CSS. Staraj się unikać nadmiernego zagnieżdżania selektorów i definiowania zbyt dużych arkuszy styli.
